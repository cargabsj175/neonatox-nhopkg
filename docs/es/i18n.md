# Internacionalización (i18n)

nhopkg utiliza **gettext** para la traducción de mensajes. Todos los mensajes visibles para el usuario en el binario principal `nhopkg` y las herramientas complementarias están envueltos en cadenas traducibles `$"..."`.

## Flujo de trabajo de traducción

### 1. Extraer cadenas traducibles

```bash
# Generar o actualizar la plantilla POT
find src/ -name '*.in' -exec xgettext --from-code=UTF-8 -o po/nhopkg.pot {} +
```

### 2. Crear una nueva traducción

```bash
msginit -i po/nhopkg.pot -l <locale> -o po/<locale>.po
```

### 3. Actualizar una traducción existente

```bash
msgmerge -U po/<locale>.po po/nhopkg.pot
```

### 4. Compilar traducciones

```bash
msgfmt -o po/<locale>.mo po/<locale>.po
install -Dm644 po/<locale>.mo /usr/share/locale/<locale>/LC_MESSAGES/nhopkg.mo
```

## Configuración

| Variable | Valor por defecto | Descripción |
|---|---|---|
| `NHOPKG_GETTEXT` | `yes` | Habilitar traducciones gettext (`yes` o `no`) |
| `TEXTDOMAIN` | `nhopkg` | Dominio de texto de gettext (no editar) |
| `TEXTDOMAINDIR` | `/usr/share/locale` | Directorio de locale (no editar) |

## Locales soportados

| Locale | Archivo | Estado |
|---|---|---|
| `es_ES.UTF-8` | `po/es.po` | Activo |
| `pt_BR.UTF-8` | `po/pt_BR.po` | Activo |

## Añadir nuevas cadenas traducibles

Al añadir nuevos mensajes en los archivos fuente, envuélvalos en `$"..."`:

```bash
echog $"Installing package %s" "$pkg"
```

Después de añadir cadenas, regenere el POT y actualice todos los archivos PO (pasos 1 y 3 anteriores).

## Notas

- `TEXTDOMAIN` y `TEXTDOMAINDIR` se establecen en `nhopkg.conf` y no deben ser modificados por los usuarios finales.
- Si `NHOPKG_GETTEXT=no`, los mensajes se muestran en inglés sin búsqueda de traducción.
- Meson ejecuta `msgfmt` durante la compilación si gettext está disponible.
