# Internationalization (i18n)

nhopkg uses **gettext** for message translation. All user-facing messages in the main `nhopkg` binary and companion tools are wrapped in `$"..."` translatable strings.

## Translation workflow

### 1. Extract translatable strings

```bash
# Generate or update POT template
find src/ -name '*.in' -exec xgettext --from-code=UTF-8 -o po/nhopkg.pot {} +
```

### 2. Create a new translation

```bash
msginit -i po/nhopkg.pot -l <locale> -o po/<locale>.po
```

### 3. Update an existing translation

```bash
msgmerge -U po/<locale>.po po/nhopkg.pot
```

### 4. Compile translations

```bash
msgfmt -o po/<locale>.mo po/<locale>.po
install -Dm644 po/<locale>.mo /usr/share/locale/<locale>/LC_MESSAGES/nhopkg.mo
```

## Configuration

| Variable | Default | Description |
|---|---|---|
| `NHOPKG_GETTEXT` | `yes` | Enable gettext translations (`yes` or `no`) |
| `TEXTDOMAIN` | `nhopkg` | Gettext text domain (do not edit) |
| `TEXTDOMAINDIR` | `/usr/share/locale` | Locale directory (do not edit) |

## Supported locales

| Locale | File | Status |
|---|---|---|
| `es_ES.UTF-8` | `po/es.po` | Active |
| `pt_BR.UTF-8` | `po/pt_BR.po` | Active |

## Adding new translatable strings

When adding new messages to source files, wrap them in `$"..."`:

```bash
echog $"Installing package %s" "$pkg"
```

After adding strings, regenerate the POT and update all PO files (step 1 and 3 above).

## Notes

- `TEXTDOMAIN` and `TEXTDOMAINDIR` are set in `nhopkg.conf` and should not be modified by end users.
- If `NHOPKG_GETTEXT=no`, messages are displayed in English without translation lookup.
- Meson runs `msgfmt` during build if gettext is available.
