# Internacionalização (i18n)

O nhopkg usa **gettext** para tradução de mensagens. Todas as mensagens voltadas ao usuário no binário principal `nhopkg` e ferramentas complementares são envolvidas em strings traduzíveis `$"..."`.

## Fluxo de tradução

### 1. Extrair strings traduzíveis

```bash
# Gerar ou atualizar o modelo POT
find src/ -name '*.in' -exec xgettext --from-code=UTF-8 -o po/nhopkg.pot {} +
```

### 2. Criar uma nova tradução

```bash
msginit -i po/nhopkg.pot -l <locale> -o po/<locale>.po
```

### 3. Atualizar uma tradução existente

```bash
msgmerge -U po/<locale>.po po/nhopkg.pot
```

### 4. Compilar traduções

```bash
msgfmt -o po/<locale>.mo po/<locale>.po
install -Dm644 po/<locale>.mo /usr/share/locale/<locale>/LC_MESSAGES/nhopkg.mo
```

## Configuração

| Variável | Padrão | Descrição |
|---|---|---|
| `NHOPKG_GETTEXT` | `yes` | Ativar traduções gettext (`yes` ou `no`) |
| `TEXTDOMAIN` | `nhopkg` | Domínio de texto do gettext (não editar) |
| `TEXTDOMAINDIR` | `/usr/share/locale` | Diretório de locale (não editar) |

## Locales suportados

| Locale | Arquivo | Status |
|---|---|---|
| `es_ES.UTF-8` | `po/es.po` | Ativo |
| `pt_BR.UTF-8` | `po/pt_BR.po` | Ativo |

## Adicionando novas strings traduzíveis

Ao adicionar novas mensagens aos arquivos fonte, envolva-as em `$"..."`:

```bash
echog $"Instalando pacote %s" "$pkg"
```

Após adicionar strings, regere o POT e atualize todos os arquivos PO (etapas 1 e 3 acima).

## Notas

- `TEXTDOMAIN` e `TEXTDOMAINDIR` são definidos em `nhopkg.conf` e não devem ser modificados por usuários finais.
- Se `NHOPKG_GETTEXT=no`, as mensagens são exibidas em inglês sem consulta de tradução.
- O Meson executa `msgfmt` durante a compilação se o gettext estiver disponível.
